Flovi Driver — Design Section 3 of 3

Screens, Error Handling & Testing Strategy

Screen Inventory

Screen Route Key Elements
Login /login Flovi logo, "Sign in with Google" button, loading state
Available Jobs /available-jobs Scrollable job cards (PENDING), pull-to-refresh, empty state
Booking Sheet (bottom sheet) Job details, CONFIRM button, loading spinner, error message
My Jobs /my-jobs Tabbed view: IN_PROGRESS / COMPLETED / CANCELLED, pull-to-refresh
Side Drawer (overlay) Avatar, driver name, nav links, Sign Out

Error Handling

Network / API errors
→ Inline error widget on list screens with "Retry" button
→ Snackbar for booking failures (non-blocking)

Auth errors
→ Google sign-in cancelled → stay on login screen silently
→ Session expired → redirect to /login via go_router redirect

Empty states
→ "No available jobs right now" illustration on Available Jobs
→ "No booked jobs yet" on My Jobs

Testing Strategy

Type What's Tested Tool
Unit Relocation model, JSON parsing, provider logic flutter test
Widget Job card, booking sheet, login button rendering flutter test
Integration Full auth flow, browse → book → appear in My Jobs flutter_test + mockito

Testing agent will run flutter test --coverage and report results.
