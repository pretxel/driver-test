Flovi Driver — Full App Design

Section 1 of 3: Architecture & Navigation. Review each section and approve before we write the spec.

Tech Stack

Framework Flutter 3 (Dart)
Authentication Supabase (Google OAuth via supabase_flutter)
State Management Riverpod 2 (flutter_riverpod + riverpod_annotation)
Navigation go_router (side drawer, named routes)
HTTP Client Dio + Supabase JWT interceptor
Theme Dark Navy + Blue (ThemeData.dark() customized)

Navigation Flow

App Start
↓ Check Supabase session
├─ No session → /login (Google Sign-In)
└─ Has session → /available-jobs (home)

Side Drawer (always accessible after login)
├─ 🔍 Available Jobs → /available-jobs
├─ 📁 My Jobs → /my-jobs
└─ Sign Out → clears session → /login

From Available Jobs
└─ Tap job → confirmation dialog → PUT API → update state → show in My Jobs

Feature → Teammate Assignment

Feature Teammate Folder
Google Auth + Routing auth-agent lib/features/auth/ + lib/core/
Browse Pending Relocations jobs-agent lib/features/available_jobs/
One-Tap Booking booking-agent lib/features/available_jobs/ (booking flow)
My Jobs View my-jobs-agent lib/features/my_jobs/
